Ты получаешь два документа: Review (факты за период) и Insights (аналитика).
На их основе подготовь два текста.

---

## 1. Отчёт для руководства

Формат: 5–10 предложений, plain text. Без markdown-разметки, без списков.
Тон: деловой, сухой, по фактам.

Структура:
- Первое предложение: период + команда + количество дейликов.
- Что сделано: ключевые результаты (2–3 самых значимых, без перечисления всех задач).
- Риски: просроченные поручения, системные блокеры, WIP-перегрузка — только если есть.
- Прогноз/рекомендация: одно конкретное действие, которое снизит главный риск.
- Последнее предложение: общая оценка (команда в норме / есть проблемы / нужно внимание).

Чего НЕ должно быть:
- Номеров задач (TASK-XXX, PR #XXX) — руководству это не нужно.
- Перечисления всех блокеров — только системные.
- Деталей реализации.
- Воды, вводных фраз, "в целом команда работала продуктивно".

---

## 2. Саммари для команды

Формат: короткое сообщение для Slack/Telegram. 10–15 строк максимум.
Тон: дружелюбный, конкретный.

Структура:
- Заголовок: период.
- Блок "Сделали": 3–5 ключевых результатов (можно с номерами задач).
- Блок "Внимание": открытые поручения с владельцами и сроками. Только просроченные или на сегодня.
- Блок "На этой неделе/дальше": 1–2 приоритета на ближайший период.
- Если есть повторяющийся блокер — упомянуть отдельной строкой.

Чего НЕ должно быть:
- Длинных описаний.
- Метрик вроде "velocity растёт" — команда это и так видит.
- Повторения всего, что было в review.
