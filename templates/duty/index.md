# Дежурство

## Что такое дежурство
Дежурство — особый тип рабочего дня, когда разработчик работает не по бэклогу,
а по приоритетам сопровождения. Дежурный следит за работой системы на всех
стендах, разбирает инциденты и проблемы, готовит релизы.

## Приоритеты работы

| № | Приоритет | Где |
|---|---|---|
| P1 | Инциденты, проблемы, разборы | ПРОМ |
| P2 | Проблемы на тестовых стендах | DEV, тестовые стенды |
| P3 | Проактивный анализ логов | все стенды |
| P4 | Сборка нового релиза | — |
| P5 | Текущие задачи | свой бэклог |

**Правило:** дежурный работает по приоритетам сверху вниз.
P(N) берётся, когда нет активных P(N-1).

## Когда что делать

### В начале дня
Если нет активных P1/P2 — начинаем с **P3** (анализ логов) и **P4** (сборка релиза).
Важные переходящие задачи с прошлого дня обычно приходят как встречи в календаре.

### В течение дня
- **P1 прилетел** → бросаем всё, идём разбирать.
- **P2 прилетел** → берём после закрытия P1.
- **P3** → делаем, когда нет активных P1/P2.
- **P4** → по графику релизов команды.
- **P5** → если всё выше закрыто.

### В конце дня
- Оформить отчёты по P1/P2/P3.
- Запушить в репозиторий отчётов дежурства.

## Отчёт дежурного

Отчёт пушится в репозиторий отчётов дежурства: **git.bb.com** (TODO: финальный URL).

**Структура:**

```
<repo>/
  YYYY-MM-DD/
    p1.md   # инциденты на ПРОМ (если были)
    p2.md   # проблемы на тестовых стендах (если были)
    p3.md   # проактивный анализ логов
```

- Имя папки — дата дежурства в формате `YYYY-MM-DD`.
- В шапке каждого файла отчёта указывается поле **Дежурный**.
- По **P4** отчёт не требуется — результат виден в Jenkins / в релизных ветках.
- По **P5** отчёт не требуется — работа по бэклогу фиксируется как обычно.
