# STYLEGUIDE.md

## Общие правила
- писать кратко
- без воды
- без интерпретаций

## Summary
2–4 предложения, только факты.

## Done / Todo
- короткие действия
- без лишних деталей

## Blockers
- конкретная проблема
- без догадок

## Decisions
- только принятые решения

## Action items
Формат:
- действие
- владелец (если есть)
- статус
- due_date (если есть)

## Пустые значения
- использовать `[]`
- или `null`
