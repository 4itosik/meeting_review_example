# AGENTS.md

## Назначение
Ты обрабатываешь протоколы встреч и создаёшь структурированные данные.

## Обязательные правила
1. Не выдумывай данные
2. Не добавляй информацию вне `raw.md`
3. Всегда создавай:
   - `summary.md`
   - `structured.json`
4. JSON должен быть валидным по `schemas/daily_meeting.schema.json`
5. Все выводы основаны только на `raw.md`

## Что фиксировать
- `done` (сделано)
- `todo` (задачи)
- `blockers` (препятствия)
- `decisions` (решения)
- `action_items` (поручения)
- `offtopic` (оффтоп)

## Action items
Добавляй только если:
- понятно, что делать
- указан владелец (если есть)

Статус `action_items.status`:
- `open`
- `in_progress`
- `done`
- `dropped`

## Блокеры
Только реальные препятствия, мешающие работе.

## Оффтоп
Не смешивать с рабочей частью.

## Использование примеров
Ориентируйся на `examples/` как на эталон формата.
