# Generate Review — weekly/monthly обзоры

## Что делает скилл

Запускает `scripts/generate_review.py` и генерирует агрегированный обзор:
1. Итог периода (задачи, блокеры, решения, поручения)
2. Что сделано — по каждому участнику
3. Блокеры и повторяющиеся блокеры
4. Решения
5. Открытые поручения
6. Ключевые темы
7. Статистика участия

## Использование

### Шаг 1: Определить параметры

- `team` — ОБЯЗАТЕЛЬНО. Спроси, если не указана.
- Период — один из вариантов:
  - `--week 2026-W13` (ISO-неделя)
  - `--month 2026-03`
  - `--from 2026-03-24 --to 2026-03-31` (произвольный диапазон)

### Шаг 2: Запустить скрипт

```bash
python3 scripts/generate_review.py --team TEAM_NAME --week YYYY-Www [--save]
python3 scripts/generate_review.py --team TEAM_NAME --month YYYY-MM [--save]
python3 scripts/generate_review.py --team TEAM_NAME --from YYYY-MM-DD --to YYYY-MM-DD [--save]
```

Флаг `--save` сохраняет обзор в `reviews/weekly/` или `reviews/monthly/`.

### Шаг 3: Показать и дополнить

Выведи обзор пользователю. Предложи:

- Добавить раздел “Выводы” — на основе паттернов в данных (повторяющиеся блокеры, просроченные items)
- Сохранить через `--save`, если ещё не сохранён

## Зависимости

- `scripts/generate_review.py`
- Наличие `meetings/` с обработанными `structured.json`
